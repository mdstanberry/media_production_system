## Use Case 1 — Script Assembly

**Trigger:** After the Stage 1 Content Brief is complete and saved in Notion.

**Folder contents to read:**
- `_COWORK_INSTRUCTIONS.md` (this file)
- Any Content Brief notes or outline documents saved in this project folder
- Any reference documents in `assets/reference/`

**Exact output requested:**

Produce a first-draft Performance Script for on-camera delivery based on the Content Brief:
- Format as speaking bullets — short, conversational, easy to scan while recording
- Each bullet = one complete thought (1–2 sentences max when spoken aloud)
- Add ANCHOR PHRASES: key sentences Dean should deliver nearly verbatim (mark as `ANCHOR: ...`)
- Add TRANSITION LINES between sections
- Flag any technical FM/CRE terms that need plain-language alternatives for CRE executive viewers (mark as `[JARGON: suggestion]`)
- Add [PAUSE] markers where a point should land before continuing
- Add [DEMO] or [SLIDE] markers where screen content or visuals take over
- Format with clear segment headers matching the Content Brief outline

## Use Case 2 — SEO Package Assembly

**Trigger:** After the video has been exported and the transcript is saved in this project folder.

**Folder contents to read:**
- `_COWORK_INSTRUCTIONS.md` (this file)
- The transcript file (any `.txt` or `.md` file in this folder containing the transcript)
- The Content Brief (if saved locally in this folder)

**Exact output requested:**

Generate the full Stage 8 SEO and metadata package for YouTube:

1. **TITLE OPTIONS** (5 variants) — optimized for search + click-through rate, under 60 characters each; label as [Search-led], [Curiosity-led], [Outcome-led], [Authority-led], [Urgency-led]
2. **DESCRIPTION** (full) — opening paragraph (2–3 sentences, appears above "Show more"), body covering key topics, CTA with subscribe prompt, chapter markers as `[TIMECODE] — Section Title`, and 3–5 hashtags at the end
3. **TAGS** (15–20) — mix of broad, mid-tail, and long-tail keywords; include FM, CRE, PropTech, and topic-specific terms
4. **CHAPTER LIST** — based on content structure, formatted for YouTube chapters
5. **CARDS & END SCREEN NOTES** — suggest 2–3 moments in the video where a card to a related topic would perform well

## Use Case 3 — Repurposing Package

**Trigger:** After the video has been published and the transcript is saved in this project folder.

**Folder contents to read:**
- `_COWORK_INSTRUCTIONS.md` (this file)
- The transcript file (any `.txt` or `.md` file in this folder containing the transcript)
- Note the Repurposing Potential tags assigned to this project in Stage 0.4 — only generate sections that match

**Exact output requested:**

Generate the Stage 10 repurposing content for all applicable tags:

**BLOG / ARTICLE** *(if tagged)*
- 800–1200 word article draft, 3rd person, authoritative tone
- Use the video transcript as the sole source — do not add outside information
- Include section headers matching the video structure

**LINKEDIN POST** *(if tagged)*
- 150–250 words, first person, professional but direct
- Open with the core insight — not "I just published a video"
- End with a question to drive comments
- Include a placeholder for the video link

**NEWSLETTER SNIPPET** *(if tagged)*
- 100–150 word summary for an email newsletter
- Focus on the "so what" for FM/CRE readers
- Include a placeholder for the video link

**YOUTUBE SHORTS** *(if tagged)*
- 2–3 standalone Short scripts, 30–60 seconds each
- Each must open with a hook, deliver one key insight, end with a prompt to engage
- Note which segment of the original video each Short derives from
