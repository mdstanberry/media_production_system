# tasks/

This directory contains the PRD (Product Requirements Document) and task list files for each layer of the Media Production System build.

## File Naming Convention

Each system layer has two paired files:

- `prd-layer-N-[name].md` — the requirements document describing what to build and why
- `tasks-layer-N-[name].md` — the actionable task list used to implement the PRD

## Files in This Directory

| File | Purpose |
|------|---------|
| `prd-layer-1-documentation-config.md` | PRD for Layer 1: documentation and configuration files |
| `tasks-layer-1-documentation-config.md` | Task list for Layer 1 |
| `prd-layer-2-notion-infrastructure.md` | PRD for Layer 2: Notion database infrastructure |
| `tasks-layer-2-notion-infrastructure.md` | Task list for Layer 2 |
| `prd-layer-3-automation-spec.md` | PRD for Layer 3: Make automation scenario spec |
| `tasks-layer-3-automation-spec.md` | Task list for Layer 3 |

## How to Use

1. Read the PRD file for a layer to understand the requirements.
2. Open the matching task list file and work through tasks one sub-task at a time.
3. Mark each sub-task complete (`[x]`) as you finish it.
